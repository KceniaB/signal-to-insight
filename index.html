// 02 — methodology

Analysis Pipeline

A reproducible end-to-end workflow from raw sensor data to insight, applicable across all project modules.

01

Data Ingestion & QC

Load raw signals (CSV/EDF), inspect missing values, validate timestamps, check sampling rates. Flag artifacts early.

02

Signal Preprocessing

Bandpass / notch filtering (Butterworth), baseline wander removal, resampling, normalization. Visualize before/after PSD.

03

Feature Extraction

Window-based statistical features (mean, std, IQR, energy, zero-crossings), spectral features (PSD bands), and non-linear HRV measures (RMSSD, pNN50, SampEn).

04

Unsupervised Exploration

K-Means clustering, PCA visualization, Isolation Forest for anomaly detection. Understand structure before adding labels.

05

Supervised Classification

Random Forest, SVM, and gradient boosting with leave-one-subject-out cross-validation. F1, confusion matrices, and feature importance.

06

Interpretation & Visualization

Publication-quality figures, interactive dashboards, and reproducible notebooks. All code open-source on GitHub.

Wearable EDA Sleep
MMASH: Activity & Sleep Exploration

The MMASH dataset (PhysioNet) contains multi-day recordings from 22 healthy participants: triaxial accelerometry, RR intervals, sleep diaries, and activity labels. This module performs a systematic EDA — behavior time distributions, overnight RR interval dynamics, and windowed HRV (RMSSD) estimation to establish baseline physiological patterns.

DATASET

MMASH — 22 subjects, 2-day recordings

SIGNALS

RR intervals, accelerometer (Actigraph), sleep diary

HRV METRIC

RMSSD computed over sliding windows

LABELS

12 activity types (sleeping → vigorous activity)

ML Classification
Activity Detection: Supervised vs Unsupervised

Binary activity classification (active vs. not active) comparing three paradigms. K-Means (k=2) and Isolation Forest represent the unsupervised approach — no labels required. Random Forest and SVM are trained with ground-truth activity annotations using 5-fold stratified cross-validation. Random Forest achieves F1 = 0.93, while K-Means reaches ~0.56, highlighting the value of labeled data even for a simple binary task.

FEATURES

Activity code, duration, accelerometer stats

UNSUPERVISED

K-Means (k=2), Isolation Forest, PCA vis

SUPERVISED

Random Forest (100 trees), SVM (RBF kernel)

EVALUATION

F1 score, confusion matrix, feature importance

ECG PPG DSP
ECG & PPG Signal Processing

Raw physiological signals from wearable sensors are corrupted by motion artifacts, baseline wander, and powerline interference. This notebook implements a full preprocessing chain: 4th-order Butterworth bandpass filter (0.5–40 Hz), Welch PSD estimation, and R-peak detection (Pan-Tompkins inspired). PPG peaks are detected for heart rate derivation from optical sensors.

FILTER

4th-order Butterworth bandpass 0.5–40 Hz

SPECTRUM

Welch PSD (scipy.signal.welch)

R-PEAKS

scipy.signal.find_peaks + distance constraint

PPG

Peak detection → instantaneous HR

Sleep HRV
Sleep Staging & Cardiac Dynamics

Sleep architecture profoundly shapes cardiac autonomic activity. This module reconstructs hypnograms from wearable data, plots overnight HR trajectories, and computes HR distributions per sleep stage (Wake, N1, N2, N3/Deep, REM). Deep sleep (N3) is associated with the lowest heart rates and highest vagal tone.

HYPNOGRAM

5-stage sleep staging (W/N1/N2/N3/REM)

HR PROFILE

Rolling mean overnight HR curve

RMSSD

Windowed RMSSD as vagal tone proxy

STAGE HR

Boxplot distribution per sleep stage

Sports Biomechanics 50 Years
Running Shoe Drop vs Race Performance (1975–2024)

50 years of running shoe design data (heel-to-toe drop, midsole cushion thickness) correlated against world records in the marathon, half marathon, 10K, 5K, 1500m, and 400m. The analysis captures three eras: the maximalist 1970s/80s, the minimalist wave triggered by Born to Run (2009), and the carbon-plate super-shoe revolution (Vaporfly, 2016+) — which is strongly correlated with the most dramatic WR improvements in road running history.

SHOE DATA

Drop height (mm), cushion thickness, carbon plate era flags

RECORDS

Marathon, HM, 10K, 5K, 1500m, 400m WR progression

ANALYSIS

Pearson correlation matrix, scatter trend lines

ERA FLAGS

Born to Run (2009), Vaporfly launch (2016)

Audio Sports Cadence
Music BPM × Running Pace: Finding Your Perfect Tempo

Music tempo acts as an external pacemaker — runners unconsciously synchronise their stride to the beat. This module models the full pace → cadence → BPM relationship, maps genre BPM distributions to running pace zones, simulates cadence entrainment under different music conditions, and provides a recommend_bpm() function that outputs the ideal music BPM range for any target pace.

MODEL

Pace → cadence → BPM via stride length estimation

GENRES

10 genres mapped by BPM distribution to pace zones

ENTRAINMENT

Cadence synchronisation simulation (coupling model)

RECOMMENDER

recommend_bpm(pace) → ideal BPM + genre matches
